# Design Method — how we get considered, non-generic UI here

This exists so the quality of work like the Think-Winners hero is **repeatable, not a fluke.**
Run this loop for any non-trivial UI (a landing, a hero, a dashboard, a key flow). It is the
"how"; [authentic-design.md](authentic-design.md) is the "what to avoid" and
[process-and-ui-dod.md](process-and-ui-dod.md) is the "done" bar.

## The loop

1. **Brief before pixels.** Write down, first: *what is this surface for, who is the audience,
   what must it communicate, what is the one action?* If the source material is ambiguous — or is
   secretly written for a different audience (the landing's source doc turned out to be a pitch to
   a **candidate**, not members) — surface that and resolve it *before* designing. Design serves the
   answer; it does not precede it.
2. **Diverge — real options, not variations.** Produce several **genuinely distinct** concepts
   (different mechanism / layout / palette), on a **real preview route, running and screenshotted** —
   never described-only. Include at least one **light/white** direction; don't default to one theme
   (see [[dont-default-to-green-palette]]).
3. **Converge with reasons.** Rank against the brief — *captivating × communicates the subject ×
   distinct* — give a **single recommendation and the counter-case**. Decide on artifacts, not adjectives.
4. **Ground every choice in the subject.** Each notable decision ties to *this* movement/brand: the
   map = statewide reach, the motto, green+gold, real Nigerian imagery (never misrepresenting stock
   people as members), tabular numerals for data. Test: *"Could this screen be lifted, unchanged,
   into an unrelated app?"* If yes, rework it.
5. **Hunt the tells.** Actively remove AI/template signals — eyebrow kickers, everything-centered,
   generic stock, purple gradients, glass-everywhere. See [authentic-design.md](authentic-design.md).
6. **Raise the bar on demand.** When asked to go higher: **research current high-craft patterns**,
   name the techniques you're applying, and add *real craft* (grade, grain, restrained motion,
   editorial type) — not more decoration.
7. **Show & sign off.** Desktop + mobile, light + dark, verified running; then get **explicit
   approval before committing** ([[ui-needs-user-signoff-before-commit]]).

## What actually keeps the bar high (the honest part)

Not inspiration. Three things:

- **Durable instructions read every session** — this doc, `authentic-design.md`, and cross-session
  memory. That's why they're written down, not left to recall.
- **Rituals that force the good steps** — the brief, the divergence, the tells-check, the sign-off.
  A good step that's optional gets skipped under time pressure; make it part of "done."
- **A high bar held out loud.** The Think-Winners jump came from the user pushing *"go higher,"*
  asking *"what is this really for,"* insisting on comparable options, and naming tells. This method
  encodes those prompts as **defaults**, so quality doesn't depend on someone remembering to ask.
